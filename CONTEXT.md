# Day of Data Baton Rouge Website

This context covers the public website for SQL Saturday / Day of Data Baton Rouge. It exists to preserve the event's public information while giving organizers more control over presentation and sponsor operations.

## Language

**Content Baseline**:
The set of public pages, copy, assets, and outbound links currently visible on the live site that phase one of the migration must preserve.
_Avoid_: Existing site, old site, current stuff

**Event Year**:
The Baton Rouge event edition for a specific calendar year, with its own date, venue, agenda, registration details, sponsors, and related public content.
_Avoid_: Season, release, version

**Canonical Domain**:
The primary public domain for the site and the destination to which any legacy domains should redirect.
_Avoid_: Mirror, alternate URL

**Sponsor**:
An organization that financially supports an Event Year in exchange for the benefits defined by a sponsor package and public recognition on the site.
_Avoid_: Vendor, partner, donor

**Sponsor Package**:
A named sponsorship offering with a defined price and benefit set that a Sponsor can select for an Event Year.
_Avoid_: Tier, deal, option bundle

**Invoice Request**:
The path a Sponsor uses when they want to pay outside immediate checkout, typically to support accounts payable, check, or ACH workflows.
_Avoid_: Offline payment, paper billing

**Sponsor Listing**:
The public display of a Sponsor on the website for an Event Year, including its logo, link, and any package-based placement.
_Avoid_: Badge, placement asset

**Publication PR**:
The pull request that proposes a website change for an Event Year and must be merged before the public site is updated.
_Avoid_: Auto-publish, direct site edit

**Sponsor Intake**:
The information a Sponsor provides so the organizers can bill them correctly and create the Sponsor Listing for the relevant Event Year.
_Avoid_: Lead form, submission

**Sponsor Case**:
The private internal record used to track a Sponsor from intake through invoicing, payment follow-up, and publication decisions.
_Avoid_: Ticket, email thread

**Archive**:
The public listing of prior Event Years and their pages once they are no longer the current event.
_Avoid_: History dump, old site

**Package Inventory**:
The number of Sponsor Listings available for a Sponsor Package in a specific Event Year.
_Avoid_: Slots, capacity
